# Hyperlink Engine

> AI-Powered Hyperlink Automation & Validation Engine for regulatory dossier submissions.
> Detects reference patterns across thousands of CTD documents, injects hyperlinks into Word and PDF renditions, validates link integrity, and reports submission readiness.

---

## Status

**Phase 1 — complete (Weeks 1–4) — `v0.1.0-phase1-complete`**

What works today:
- Full Layer 1 ingestion: Word, PDF, eCTD backbone, Dossplorer mock
- Full Layer 2 parsing: run-level styling, PDF spans/blocks/links, leaf manifest
- Full Layer 3 detection cascade: 12 regex patterns + spaCy NER (rule fallback) + local-only LLM disambiguation (Ollama or deterministic stub)
- Full Layer 4 injection: Word hyperlinks **and** bookmarks, PDF link annotations, Dosscriber-aware style preservation, in-module eCTD xref scaffold
- Layer 5 baseline validation: existence checker (offline) + target-correctness validator (token-Jaccard or sentence-transformers)
- Reporting: stable CSV exporter for per-doc and dossier-wide reports
- 189 unit tests, 83% coverage across the package
- Phase 1 acceptance run: **20 docs / 1,255 hyperlinks / 0 broken / detection F1 = 1.000**

What's coming:
- Weeks 5–8 (Phase 2) — eCTD backbone graph, cross-module linking, Celery bulk pipeline, anomaly detection, Streamlit dashboard
- Weeks 9–12 (Phase 3) — viewer compatibility, HA rules, GxP audit, live Dossplorer integration, React production dashboard

## Quickstart

### 1. Install

```powershell
# From the hyperlink-engine/ directory
poetry install
```

> If you don't have Poetry yet: https://python-poetry.org/docs/#installation
> If you only have pip: `pip install -e .` should work for the Phase 1 spike subset.

### 2. Generate synthetic test data

```powershell
poetry run python -m scripts.bootstrap_synthetic_data --out data/synthetic --docs 20
```

This produces 20 mock CTD documents under `data/synthetic/` with ~500 embedded references plus a mock `index.xml` eCTD backbone.

### 3. Run the W1.5 spike

```powershell
# Inspect detections (no file changes)
poetry run hyperlink-engine inspect --input data/synthetic/m2/2-5-clin-overview/2-5-clin-overview.docx

# Link the document end-to-end
poetry run hyperlink-engine spike `
    --input data/synthetic/m2/2-5-clin-overview/2-5-clin-overview.docx `
    --output output/2-5-clin-overview_linked.docx `
    --report output/2-5-clin-overview_report.csv
```

Open the output `.docx` in Word — every Section/Table/Figure/Listing/Module/Study-ID reference should be a clickable hyperlink.

### 4. Run the test suite

```powershell
poetry run pytest
```

Coverage gate is set to **85%** in `pyproject.toml`.

## Configuration

Settings live in `src/hyperlink_engine/config/settings.py` and can be overridden via environment variables (prefix `HYPERLINK_`) or a `.env` file. Copy `.env.example` to `.env` and adjust.

Key knobs:
- `HYPERLINK_OLLAMA_HOST` — local LLM endpoint (default `http://localhost:11434`)
- `HYPERLINK_ENFORCE_LOCAL_LLM_ONLY=true` — refuses any non-localhost LLM call (required for GxP / 21 CFR Part 11 compliance)
- `HYPERLINK_LOG_FORMAT=console` — switch from JSON to colored console logs during dev

## Compliance posture

- All AI inference is **local-only** (Ollama + sentence-transformers). No external API calls.
- Documents are never mutated in place — every output is a new file.
- Every link injection appends an immutable audit record to `audit.jsonl`.
- Source-of-truth for design decisions: [`docs/adr/`](docs/adr/).

See plan §8.1 and ADR-0004 for the full GxP posture.
